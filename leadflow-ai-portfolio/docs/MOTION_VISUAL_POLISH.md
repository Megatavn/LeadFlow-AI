# Motion & Visual Polish Notes

LeadFlow AI v2.5 focuses on making the app feel closer to a real mobile product while keeping the technical scope lightweight.

## Motion rules

- Use short transitions: 160–340ms.
- Prefer fade/slide/spring-like motion over decorative effects.
- Keep motion functional: tab changes, bottom sheets, button presses, toast feedback.
- Respect `prefers-reduced-motion`.

## Visual rules

- Use illustrations only where they improve comprehension: onboarding, empty states, hero area.
- Avoid heavy image assets so the APK stays light.
- Use generated avatars to reduce text-heavy CRM screens.
- Maintain clear hierarchy: score, probability, health, next action, then details.

## Portfolio value

These upgrades demonstrate:

- Mobile-first UX judgment
- Product polish thinking
- Micro-interaction design
- Offline-first implementation
- Public GitHub presentation quality
